◆ unregisterAction()

void CuPoller::unregisterAction ( CuTangoActionI *  a)

unregister an action from the poller.

Parameters
athe action to unregister

A CuTangoActionI can invoke unregisterAction from within CuTangoActionI::onResult. In this case, the operation is delayed after CuPoller::onResult has finished delivering results. If the method is called when CuPoller is not inside CuPoller::onResult, the action is unregistered immediately.

  • after the last action is removed, this object is removed from the CuActivityManager thread listeners before next step:
  • a CuRemovePollActionEvent is posted to the CuPollingActivity managing the CuTangoActionI device with the CuPoller::period interval
  • a dummy CuData with the "exit" flag is delivered through CuTangoActionI::onResult. This is why unregister operations must be enqueued and performed outside result delivery taking place in CuPoller::onResult
