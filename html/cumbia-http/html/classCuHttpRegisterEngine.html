CumbiaHttp * CuHttpRegisterEngine::registerWithDefaults ( CumbiaPool *  cu_pool,
CuControlsFactoryPool &  fpoo 
)

Setup CumbiaHttp with some default options.

Parameters
cu_poola pointer to a valid CumbiaPool. A new CumbiaHttp instance will be registered with name "http"
fpooa reference to a CuControlsFactoryPool that will be initialized with registerImpl, name "http"
Returns
a pointer to a newly allocated CumbiaHttp registered into cu_pool with name "http"
Default operations
The following helpers are installed on CumbiaHttp:
  • CuHttpTangoReplaceWildcards to replace wildcards in the command line arguments as the native tango engine would do;
  • CuHttpTangoHelper is used to get Tango source patterns
  • CuHttpTangoSrcHelper is registered to CumbiaHttp so that attribute and command readings suitable to be monitored over time can be distinguished from single-shot database operations and the TANGO_HOST is prepended to the source name if missing. See CuHttpTangoSrcHelper::setOperation
