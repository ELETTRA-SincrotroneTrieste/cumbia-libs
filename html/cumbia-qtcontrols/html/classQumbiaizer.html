Detailed Description

QTangoizes anything (not really anything, but...)

This base class is reimplemented by QTWatcher and QTWriter to use QTangoCore in order to update widgets that are not QTango aware.

associate a QObject and one or two methods to be invoked by the class on new data arrival (QTWatcher) or on target execution (QTWriter).

This method associates a QObject on which to invoke a slot when new data arrives (for readers) or a QObject from which to wait for signals to execute a target (for writers).

Parameters
objectthe QObject pointer
methodthe SLOT (readers) or SIGNAL (writers) methods to associate
setPointSlotif not NULL, this is the slot to be invoked in auto configuration to initialize the value read first.
connTypethe Qt::ConnectionType for signals/slots

The type of input parameter is detected on the basis of the method specified. For instance setValue(int) will detect an Int data type. If you specify a setPointSlot, remember that its data type must be the same as the data type of the method.

Reimplemented in QuWriter.

◆ configure

void Qumbiaizer::configure ( const CuData & data)
protectedvirtualslot

Configures the attached object, if any.

If setAutoconfSlot has been called for some AutoConfType type, then the matching slots are invoked. If a custom slot has not been specified for the minimum and maximum properties, the method sets the minimum and maximum properties on the object, if they are defined. If attach was called specifying an additional slot to set the set point value, then it is invoked. This can be applied only to read/write quantities, when CuData contains the "w_value" value. Tango and Epics support read/write quantities.

The configured signal is emitted at the end. It can be used by the client to access other configuration parameters in data. The connectionOk signal is also emitted for convenience.

See also
setAutoconfSlot

void Qumbiaizer::setAutoConfSlot ( AutoConfType act,
const char * slot )

when minimum and maximum values are available, these slots are invoked.

associate an auto configuration type to a slot to set the auto configuration value

These slots are invoked when the auto configuration is available. The QTWatcher api provides shortcuts to set minimum, maximum, min err, min warn, units. The supported data types for slots to be invoked are the ones enumerated in AutoConfType. Slots must take exactly one argument, and must be of type
int, double, short, unsigned int, QString.
Nothing else is supported.

See also
AutoConfType
Parameters
actan AutoConfType value
aslot that will be invoked at the configuration stage

Call this one before setSource to ensure that only one read is performed.
