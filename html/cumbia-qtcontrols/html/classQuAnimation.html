Detailed Description

A helper class to draw animations on a widget. Presently, a simple animation is drawn at the bottom of a given rectangle (supposedly, the widget rect)

QuAnimation is a class drawing animations within a rectangle. A QuAnimation has a

  • type (currently, the only supported type is QuAnimation::BottomLine)
  • a duration (mapped to QVariantAnimation::duration)
  • a start and end value (mapped to the respective QVariantAnimation properties)

The underlying QVariantAnimation easing curve is set to <cite> QEasingCurve::Linear</cite> The easing curve can be changed accessing QVariantAnimation through the qAnimation call.

Bottom line animation type
The QuAnimation::BottomLine type draws a line at the botton of the given widget that shrinks and disappears within the QuAnimation::duration time in milliseconds
Ray of light animation type
A ray of light flashes all along the drawing rectangle, clockwise, from the top left corner.

QuAnimation has two helper methods which a QWidget can use to be animated:

  • clippedRect that calculates the minimum rectangle needed to draw the animation
  • draw that draws on the widget's rect according to the QuAnimation::Type

QuAnimation::start can be used to start or <cite>stop and restart</cite> the animation The progress of the animation is notified using a <cite>Qt slot</cite> named

void onAnimationValueChanged(const QVariant&)

that must be implemented within the widget

Example
QuButton::QuButton(QWidget *w, CumbiaPool *cumbia_pool, const CuControlsFactoryPool &fpool, const QString &text)
: QPushButton(w) {
d->context = new CuContext(cumbia_pool, fpool);
// ...
m_animation.installOn(this); // m_animation is a private member of type QuAnimation
}
void QuButton::onAnimationValueChanged(const QVariant &v) {
// optimize animation paint area with the clipped rect provided by QuAnimation::clippedRect
update(d->animation.clippedRect(rect()));
}
void QuButton::paintEvent(QPaintEvent *pe) {
QPushButton::paintEvent(pe);
// m_write_ok stores the success of the last operation
// performed on button click
//
if(m_animation.currentValue() != 0.0) {
m_animation.draw(this, m_write_ok ? QColor(Qt::green) : QColor(Qt::red));
}
}

QuAnimation::QuAnimation ( Type  t = ShrinkingBottomLine)

the class constructor

Parameters
tthe type of animation, to be chosen amongst QuAnimation::Type

The default animation type is QuAnimation::ShrinkingBottomLine

Default values
  • The duration of the underlying QVariantAnimation is set to 1000 milliseconds
  • The start and end values of the underlying QVariantAnimation are set to 100.0 and 0.0 respectively. They cannot be directly changed
  • the pen color used to draw lines is set to Qt::gray
  • The default animation type is QuAnimation::ShrinkingBottomLine
  • The ray length for the type QuAnimation::RayOfLight is set to 0.5, meaning half of the widget height
A QuAnimation must be first installed (see installOn) on a QWidget in order to be started/stopped/looped.

◆ clippedRect()

QRect QuAnimation::clippedRect ( const QRect &  widgetRect) const

given the widget rectangle, returns the clipped rectangle that optimizes the paint area.

Parameters
widgetRectthe rectangle of the widget to be animated
Returns
an optimized rectangle with the minimum area that involves the update region, according to the chosen QuAnimation::Type
Example
If we are using the QuAnimation::BottomLine animation, a rectangle with the input widgetRect's full width and a height of just a few pixels is returned in order to minimize the paint area. If the animation type is QuAnimation::RayOfLight: then the whole widget rect is returned.
Usage
Within the widget's onAnimationValueChanged SLOT, the returned value of this method can be used as input to the QWidget::update(const QRect&) call
Example usage
QuButton::onAnimationValueChanged slot and QuButton::paintEvent provide a working example.

References ExpandingBottomLine, and ShrinkingBottomLine.
